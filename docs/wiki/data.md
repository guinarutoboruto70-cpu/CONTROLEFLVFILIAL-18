# Data Model & Storage

The application stores its data in the browser via `localStorage`. There is no server-side persistence.

## Storage keys

- `produtos` (JSON array) ‚Äî product catalog used by autocomplete, quebra, and cost lookup.
- `quebrasDia` (JSON array) ‚Äî daily break records and tratativa outcome status.
- `vendasFLV` (JSON array) ‚Äî daily sales records.
- `baseFLVVersao` (string) ‚Äî migration/version tag for initializing the product base.
- `premiumVencimento` (string/number) ‚Äî epoch milliseconds indicating premium expiry.

See: `getProdutos/setProdutos`, `getQuebras/setQuebras`, `getVendas/setVendas` in [index.html:L512-L517](file:///workspace/index.html#L512-L517).

## Entities (shape)

### Product

Stored in `produtos`.

Typical fields (as written by `carregarBase()` and edited by `salvarProduto()`):

- `id`: normalized name identifier (uppercase, accent-insensitive) generated by `gerarId()` ([index.html:L356-L358](file:///workspace/index.html#L356-L358))
- `nome`: product name (uppercase)
- `cod`: product code (string)
- `tipo`: `"KG"` or `"UN"`
- `fator`: legacy/base factor (used as default packaging quantity)
- `valorEmbalagem`: packaging/box price input
- `qtdEmbalagem`: packaging quantity (used for cost derivation)
- `custo`: derived unit cost (string; 3 decimals in current UI)

Related logic:

- Base migration: [index.html:L519-L543](file:///workspace/index.html#L519-L543)
- Cost calculation: [index.html:L470-L497](file:///workspace/index.html#L470-L497)
- Save: [index.html:L633-L679](file:///workspace/index.html#L633-L679)

### Quebra (Break record)

Stored in `quebrasDia`.

Created/updated by `adicionarOuEditarQuebra()` ([index.html:L819-L881](file:///workspace/index.html#L819-L881)).

Fields include:

- `id`: `Date.now().toString()`
- `data`: ISO date string (`YYYY-MM-DD`)
- `produto`: product name (uppercase)
- `cod`: product code
- `quantidade`: text as entered
- `tipo`: `"KG"` or `"UN"`
- `custoUnitario`: string with 3 decimals
- `valorQuebra`: string with 2 decimals

Tratativa linkage/outcome fields (used by ‚ÄúResultado Tratativa‚Äù and monthly/dashboard):

- `marcadoTratativa`: boolean (sent to tratativa flow)
- `statusTratativa`: `"PENDENTE" | "APROVADO" | "PARCIAL" | "REPROVADO"`
- `aprovado`: boolean
- `qtdAprovada`: string with 3 decimals (only in parcial flow)
- `valorConciliado`: string with 2 decimals (approved amount)
- `quebraLiquida`: string with 2 decimals (remaining break value after approval)

### Venda (Sale record)

Stored in `vendasFLV`.

Created/updated by `salvarVenda()` ([index.html:L1256-L1288](file:///workspace/index.html#L1256-L1288)).

- `id`: `Date.now().toString()`
- `data`: ISO date string (`YYYY-MM-DD`)
- `valor`: string with 2 decimals (pt-BR parsing supported by `numeroBR`)

## Non-persisted runtime state

These values are kept in memory only (lost on refresh):

- `banco`: list of composed tratativas shown in tela7 (‚ÄúLista‚Äù) ([index.html:L334-L335](file:///workspace/index.html#L334-L335))
- `fotos`: selected photos + object URLs used for previews/sharing ([index.html:L333-L334](file:///workspace/index.html#L333-L334))
- `textoFinal`: the composed tratativa message string ([index.html:L332-L332](file:///workspace/index.html#L332-L332))

If persistence across reloads is required, the current design would need to serialize these to `localStorage` (or an external backend).
